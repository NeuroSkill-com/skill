# Огляд

## Трансляція в реальному часі
{app} передає похідні метрики ЕЕГ та статус пристрою через локальний WebSocket-сервер. Події: eeg-bands (~4 Гц — 60+ показників), device-status (~1 Гц), label-created.

## Команди
Клієнти можуть надсилати JSON-команди через WebSocket: status, calibrate, label, search, sessions, compare, sleep, umap/umap_poll. Відповіді у JSON з полем "ok".

# Довідник команд

## status
_(немає)_

Повертає стан пристрою, інформацію сесії та якість сигналу.

## calibrate
_(немає)_

Відкриває вікно калібрування. Потрібен підключений пристрій.

## label
text (рядок, обов'язковий); label_start_utc (u64, необов'язковий)

Вставляє мітку з міткою часу в базу даних.

## search
start_utc, end_utc (u64, обов'язкові); k, ef (u64, необов'язкові)

Шукає k найближчих сусідів в індексі HNSW.

## compare
a_start_utc, a_end_utc, b_start_utc, b_end_utc (u64, обов'язково)

Порівнює два часових діапазони, повертаючи агреговані метрики потужності діапазонів (відносні потужності, показники фокусу/розслаблення/залученості та FAA) для кожного. Повертає { a: SessionMetrics, b: SessionMetrics }.

## sessions
_(немає)_

Перелічує всі сесії embeddings з щоденних баз даних. Безперервні діапазони запису (розрив > 2 хв = нова сесія). Найновіші першими.

## sleep
start_utc, end_utc (u64, обов'язково)

Класифікує кожну епоху як стадію сну (Неспання/N1/N2/N3/REM). Повертає гіпнограму + підсумок.

## umap
a_start_utc, a_end_utc, b_start_utc, b_end_utc (u64, обов'язково)

Ставить у чергу завдання 3D UMAP проекції. Повертає job_id для опитування. Неблокуюче.

## umap_poll
job_id (рядок, обов'язково)

Опитує результат завдання UMAP. Повертає { status: pending | done, points?: [...] }.

## say
text: string (обов'язковий)

Промовити текст через локальний TTS. Fire-and-forget — повертається негайно. Ініціалізує рушій при першому виклику.
