# Огляд
Проактивні хуки автоматично запускають дії, коли останні патерни EEG відповідають ключовим словам або станам мозку.

## Що таке проактивні хуки?
Проактивний хук стежить за embeddings міток EEG в реальному часі. Коли косинусна відстань падає нижче порогу, хук спрацьовує — команда, сповіщення, TTS або WebSocket-подія. Дозволяє створювати автоматизації нейрозворотного зв'язку без коду.

## Як це працює
Застосунок обчислює embeddings EEG кожні кілька секунд і порівнює з ключовими словами через HNSW-індекс. Кулдаун запобігає повторному спрацюванню. Все локально — жодні дані не покидають комп'ютер.

## Сценарії
Кожен хук може бути обмежений сценарієм — Когнітивний, Емоційний, Фізичний або Всі. Когнітивний: концентрація, відволікання, розумова втома. Емоційний: стрес, спокій, фрустрація. Фізичний: сонливість, фізична втома. 'Всі' — без обмежень.

# Налаштування хука
Кожен хук має поля, що контролюють коли і як він спрацьовує.

## Назва хука
Описова та унікальна назва (напр. 'Deep Work Guard', 'Calm Recovery'). Використовується в журналі історії та WebSocket-подіях.

## Ключові слова
Ключові слова або короткі фрази для опису стану мозку (напр. 'focus', 'deep work', 'stress', 'tired'). Вбудовуються тим же sentence-transformer моделлю. Хук спрацьовує коли embeddings EEG близькі.

## Пропозиції ключових слів
При введенні застосунок пропонує терміни з історії міток — нечітке та семантичне зіставлення. Позначка джерела: 'нечіткий', 'семантичний' або обидва. ↑/↓ та Enter для прийняття.

## Поріг відстані
Максимальна косинусна відстань (0–1). Низькі значення = суворіше, високі = м'якше. Типово: 0.08 (дуже суворо) до 0.25 (м'яко). Почніть з 0.12–0.16 та налаштуйте інструментом пропозиції.

## Інструмент пропозиції відстані
Натисніть 'Запропонувати поріг' для аналізу EEG-даних. Інструмент обчислює розподіл відстаней (min, p25, p50, p75, max) та рекомендує поріг. Візуальна шкала перцентилів. 'Застосувати' для використання значення.

## Останні посилання
Кількість останніх семплів embedding для порівняння (за замовчуванням: 12). Вищі значення = згладжування піків, нижчі = швидша реакція. Діапазон: 10–20.

## Команда
Опціональний рядок команди в WebSocket-події (напр. 'focus_reset', 'calm_breath'). Зовнішні інструменти автоматизації можуть реагувати для запуску дій або скриптів.

## Текст вмісту
Опціональне повідомлення в події спрацювання (напр. 'Зробіть перерву на 2 хвилини.'). Показується в сповіщеннях та може бути озвучене через TTS.

# Додатково
Поради, історія та інтеграція із зовнішніми інструментами.

## Швидкі приклади
Готові шаблони: Deep Work Guard (когнітивна концентрація), Calm Recovery (емоційний стрес), Body Break (фізична втома). Натисніть для додавання з попередньо заповненими значеннями. Налаштуйте під ваші EEG-патерни.

## Історія спрацювань хука
Згортаний журнал, що фіксує кожне спрацювання: мітка часу, мітка, косинусна відстань, команда, ключові слова. Для аудиту поведінки, перевірки порогів та налагодження. Вбудована пагінація.

## WebSocket-події
JSON-подія транслюється через WebSocket API: назва хука, команда, текст, мітка, відстань, мітка часу. Зовнішні клієнти можуть слухати — приглушити світло, поставити музику на паузу, надіслати Slack-повідомлення тощо.

## Поради з налаштування
Почніть з одного хука та кількох ключових слів, що відповідають існуючим міткам. Використовуйте інструмент пропозиції для початкового порогу. Стежте за історією день. Знижте поріг при хибних спрацюваннях, підвищте якщо не спрацьовує. Специфічні ключові слова покращують точність.
