# Огляд
NeuroSkill включає опціональний локальний LLM-сервер — приватний AI-асистент, сумісний з OpenAI, без надсилання даних у хмару.

## Що таке функція LLM?
Функція LLM вбудовує сервер інференції llama.cpp. Він обслуговує OpenAI-сумісні кінцеві точки (/v1/chat/completions, /v1/completions, /v1/embeddings, /v1/models, /health) на тому ж порті, що й WebSocket API. Будь-який сумісний клієнт може підключитися.

## Конфіденційність та офлайн
Вся інференція локальна. Жодні дані не покидають localhost. Лише початкове завантаження моделі з HuggingFace Hub потребує інтернету. Після кешування — повністю офлайн.

## OpenAI-сумісний API
Сервер спілкується тим же протоколом, що й OpenAI API. Будь-яка бібліотека з параметром base_url (openai-python, openai-node, LangChain, LlamaIndex) працює одразу. Встановіть base_url на http://localhost:<port>/v1.

# Керування моделями
Переглядайте, завантажуйте та активуйте GGUF-квантовані мовні моделі з вбудованого каталогу.

## Каталог моделей
Каталог перелічує сімейства моделей (Qwen, Llama, Gemma, Phi) з варіантами квантування. Використовуйте випадаючий список для перегляду. Моделі з ★ — рекомендований вибір.

## Рівні квантування
Кожна модель доступна в рівнях GGUF (Q4_K_M, Q5_K_M, Q6_K, Q8_0 тощо). Нижчі квантування менші та швидші, але жертвують якістю. Q4_K_M — найкращий компроміс. Q8_0 — майже без втрат, але вдвічі більше пам'яті. BF16/F16/F32 — без квантування.

## Позначки сумісності
Кольорова позначка: 🟢 Відмінно — вміщується у VRAM з запасом. 🟡 Добре — вузький запас. 🟠 Тісно — часткове CPU-розвантаження. 🔴 Не вміщується. Враховує VRAM, RAM, розмір моделі та контекст.

## Візуальні / мультимодальні моделі
Сімейства Vision/Multimodal включають опціональний проектор (mmproj). Завантажте обидва для підтримки зображень у чаті. Проектор розширює текстову модель — не самостійна модель.

## Завантаження та видалення
Натисніть 'Завантажити' для отримання моделі з HuggingFace Hub. Прогрес-бар показує стан у реальному часі. Можна скасувати. Завантажені моделі зберігаються локально та можуть бути видалені. 'Оновити кеш' для повторного сканування.

# Налаштування інференції
Налаштуйте спосіб завантаження та виконання моделей сервером.

## Шари GPU
Кількість шарів transformer для GPU. 'Всі' для максимальної швидкості, 0 для CPU. Проміжні значення розподіляють модель між GPU та CPU — корисно коли модель трохи перевищує VRAM.

## Пристрій інференсу (CPU / GPU)
Оберіть, чи виконувати інференс LLM на CPU чи GPU. Зміна пристрою набирає чинності негайно — сервер автоматично перезапускається, якщо він вже працює. На CPU всі шари transformer виконуються на процесорі хоста (libomp використовується для паралельного BLAS). На GPU шари розвантажуються на Metal (macOS) або Vulkan (Linux/Windows). Використовуйте режим CPU, коли VRAM GPU недостатньо або коли GPU зарезервований для вбудовування ЕЕГ.

## Розмір контексту
Розмір KV-кешу в токенах. 'Авто' — за замовчуванням моделі. Більші контексти зберігають більше історії, але споживають більше пам'яті. При помилках пам'яті зменште до 4K або 2K.

## Паралельні запити
Максимум одночасних циклів декодування. Більше клієнтів можуть ділити сервер, але пікова пам'ять зростає. Для одного користувача 1 достатньо.

## API-ключ
Опціональний Bearer-токен для запитів /v1/*. Залиште порожнім для відкритого доступу на localhost. Встановіть ключ для обмеження доступу в локальній мережі.

# Вбудовані інструменти
Чат LLM може викликати локальні інструменти для збору інформації або виконання дій від вашого імені.

## Як працюють інструменти
Модель може запитувати виклик інструментів під час розмови. Застосунок виконує їх локально та повертає результат. Інструменти викликаються лише за явним запитом — ніколи у фоні.

## Безпечні інструменти
Дата, Місцезнаходження, Веб-пошук, Веб-отримання та Читання файлу — тільки для читання. Дата повертає дату/час. Місцезнаходження — IP-геолокація. Веб-пошук — DuckDuckGo. Веб-отримання — текст URL. Читання — локальні файли з пагінацією.

## Привілейовані інструменти (⚠️)
Bash, Запис файлу та Редагування файлу можуть змінити систему. Bash виконує shell-команди. Запис створює/перезаписує файли. Редагування — пошук-заміна. Вимкнені за замовчуванням з позначкою попередження. Вмикайте лише розуміючи ризики.

## Режим виконання та обмеження
Паралельний режим викликає кілька інструментів одночасно (швидше). Послідовний — по одному (безпечніше). 'Макс. раундів' обмежує цикли інструмент/результат. 'Макс. викликів за раунд' — одночасні виклики.

# Чат та журнали
Працюйте з моделлю та відстежуйте активність сервера.

## Вікно чату
Відкрийте чат з картки LLM-сервера або tray-меню. Інтерфейс з Markdown, підсвічуванням коду та візуалізацією викликів. Розмови тимчасові — не зберігаються. Моделі з підтримкою зображень приймають вкладення.

## Зовнішні клієнти
Спрямуйте будь-який OpenAI-сумісний інтерфейс на http://localhost:<port>/v1, встановіть API-ключ якщо налаштований, виберіть модель з /v1/models. Варіанти: Open WebUI, Chatbot UI, Continue (VS Code), curl/httpie.

## Журнали сервера
Переглядач транслює вивід сервера: прогрес завантаження, швидкість токенів, помилки. Режим 'Детальний' для діагностики llama.cpp. Автопрокрутка — зупиняється при ручному скролі вгору.
