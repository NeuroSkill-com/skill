## [0.0.86] — 2026-04-04

### Security

- **Harden daemon token storage**: store API tokens as salted SHA-256 hashes (`token_hash` + `token_salt`) instead of plaintext.
- **Use constant-time token comparison** during validation to reduce timing-leak risk.
- **Migrate legacy plaintext tokens** to hashed format on load.

### API / Auth

- **Expand multi-token auth** with ACL + expiry support (`admin`, `read_only`, `data`, `stream`; `week/month/quarter/never`).
- **Enforce active-token cap** (50 active tokens max) with clear conflict response.
- **Protect default token** from deletion (`/v1/auth/tokens/delete` now rejects `id=default`).
- **Support default token refresh** endpoint and in-memory token update.
- **Differentiate auth failures**: return `403 forbidden` for ACL-denied requests and `401 unauthorized` for missing/invalid tokens.

### App / UI

- **Token management UI updates** for refresh/create/revoke/delete flows and newly created token display.
- **Invalidate daemon bootstrap cache** after default-token refresh so clients pick up the new token.

### Fixes

- **Restore `detect_device_kind(...)` helper** in `src-tauri/src/lifecycle.rs` so lifecycle unit tests compile and pass.
