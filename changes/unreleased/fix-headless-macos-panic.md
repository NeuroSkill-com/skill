### Bugfixes

- **Headless browser no longer crashes the app on macOS**: On macOS, tao requires the event loop on the main thread, but Tauri already owns it. The headless browser launch (used by `web_search render=true` and `web_fetch render=true`) panicked with "EventLoop must be created on the main thread!" on a spawned thread, which propagated through tao's `extern "C"` observer callback and aborted the entire process. `catch_unwind` cannot help because the panic occurs on the spawned thread, not the caller. Fix: `Browser::launch` now checks `NSRunLoop.mainRunLoop.currentMode` via the Objective-C runtime before spawning the event-loop thread. When the main run loop is already active (Tauri), it returns `Err` immediately — no thread is spawned, no panic occurs. Both `web_search` and `web_fetch` with `render=true` gracefully fall back to plain HTTP fetch with HTML tag stripping, producing clean text content suitable for LLM consumption.
